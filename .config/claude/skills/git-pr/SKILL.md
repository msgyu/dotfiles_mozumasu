# 指示

変更がコミット済みであることを確認し、PRタイトル/本文を作成する。

## PR本文テンプレート

```
## 概要
- 変更点を1〜3行で要約

## 変更内容
- 主要な変更点を箇条書き

## テスト
- `command` (pass/fail)

## 補足
- 補足や注意点（必要な場合のみ）
```

## コマンド方針

- `gh` が使える場合は `gh pr create` を利用
- `gh` がない場合は、タイトル/本文のみ提示して手動作成を依頼

## 規律

- 未コミットの変更がある場合は先にコミットする
- テスト未実行なら `Testing` に未実行と明記する
- PRは1つの論理的変更に絞る
